# 原文句子逻辑重组

给定一段原始文字与若干 `###` h3 小节标题，将原文所有句子按照核心意思逐一剪切归类到对应小节下，一字不删，输出到根目录的新 `.md` 文件，并确保输出内容完整、无遗漏。

## 操作流程

### 第一步：理解原文与分类框架

1. 通读原文，理解每一个段落的意思。

2. 阅读用户提供的所有 `###` h3 标题，理解每个小节的核心意思。

### 第二步：逐句归类

- 逐句处理原文，每句按以下规则归类：

    - **核心意思匹配**：判断该句子在语义上最归属于哪个 h3 小节，将其划入该小节。

    - **一字不改**：原文句子原样保留，包括错别字、口语化表达及标点符号，不得删除、合并或改写任何字词。

    - **剪切而非复制**：每个句子只归入一个小节，不得重复出现。

    - **无法归类时**：若某句子与所有 h3 标题均不相关，将其单独放入文末的 `### 未归类内容` 小节。

### 第三步：输出到新文件
 
- **不添加内容**：除用户提供的小节标题外，不得向输出文件中插入任何新文本。

- **输出为新文件**：在工作区根目录创建新的 `.md` 文件（文件名：`重组输出.md`），不得直接编辑原文件。

- **文件结构：**

    - 
        ```markdown
        ### {h3 标题1}

        {归入此小节的句子}

        ### {h3 标题2}

        {归入此小节的句子}

        ...

        ### 未归类内容（如有）

        {无法归类的句子}
        ```

### 第四步：遗漏核查

- 完成归类输出后，执行以下核查：

  1. 将原文所有句子列成一个编号清单。

  2. 逐条核对：该句子是否出现在输出文件中？

  3. 若发现有遗漏，立即将遗漏句子补充到最匹配的小节末尾（或 `### 未归类内容`）。

  4. 向用户报告：「核查完成，共 N 句，输出 N 句，无遗漏」或「发现 X 句遗漏，已补充」。
