# 20260303183856. IP 地址批量分配

🟢 难度：简单

## 题目描述

### 业务场景

- **问题背景**：
    
    - 数据中心上架了一批新服务器，网络工程师需要给它们批量分配 IP 地址。每台机器只能用一个固定 IP，而且同一个网段内已经有部分 IP 被其他设备占用了——这些占用记录存在 IPAM（IP 地址管理系统）里。传统做法是工程师手工在 Excel 表格里查看哪些 IP 已被占用，然后一个一个挑选空闲 IP 填写。

- **导致的问题**：

    1. **效率低下**：50 台服务器需要手工查询和填写 50 个 IP，耗时且枯燥

    2. **容易出错**：手工挑选容易漏查已占用的 IP，导致 IP 冲突；或者不小心分配了网络地址/广播地址

    3. **难以维护**：后续新增设备时，需要重新翻查 Excel 找空闲 IP，容易出错

- **解决方案**：开发一个自动分配程序，输入一个网段、已占用 IP 清单和需要分配的数量，程序自动按顺序挑出可用 IP。这样可以：

    - 几秒钟完成几十上百台设备的 IP 分配

    - 保证 IP 连续、无冲突、无遗漏

### 核心问题

**输入**：
- 一个 CIDR 格式的网段（如 `192.168.1.0/24`）
- 一个已占用的 IP 列表
- 需要分配的设备数量 `n`

**输出**：
- 按从小到大顺序排列的 `n` 个可用 IP 地址

**约束**：
- 网段的第一个地址（网络地址）和最后一个地址（广播地址）不能分配
- 若可用 IP 不足，返回错误
- subnet 是合法的 CIDR 格式字符串，前缀长度范围为 /8 ~ /30
- used 中的每个 IP 都属于该网段，且不会包含网络地址或广播地址
- used 中没有重复 IP
- 1 ≤ n，若 n 超出可用数量，返回 error
- 返回的 IP 列表必须按从小到大的顺序排列

## 示例 1：基本分配

输入：
```json
{
  "subnet": "192.168.1.0/24",
  "used": ["192.168.1.1", "192.168.1.3"],
  "n": 4
}
```

输出：
```json
["192.168.1.2", "192.168.1.4", "192.168.1.5", "192.168.1.6"]
```

解释：
- 网段 192.168.1.0/24 的可用范围是 192.168.1.1 ~ 192.168.1.254（共 254 个）
- 已占用 .1 和 .3，跳过它们，按顺序取出 4 个空闲地址：.2、.4、.5、.6

## 示例 2：边界网段

输入：
```json
{
  "subnet": "10.0.0.0/30",
  "used": [],
  "n": 2
}
```

输出：
```json
["10.0.0.1", "10.0.0.2"]
```

解释：
- /30 网段只有 4 个地址：.0（网络地址）、.1、.2、.3（广播地址）
- 可用地址只有 .1 和 .2，没有已占用，直接分配这两个

## 示例 3：资源不足

输入：
```json
{
  "subnet": "10.0.0.0/30",
  "used": ["10.0.0.1"],
  "n": 2
}
```

输出：
```json
{
  "error": "没有足够的可用 IP，需要 2 个，但只剩 1 个"
}
```

解释：
- /30 只有 2 个可用地址（.1 和 .2），.1 已占用，只剩 .2 一个，不够分配 2 台设备，返回错误

## 相关知识点

- CIDR 表示法与子网掩码的换算
- 将 IPv4 字符串转为 `uint32` 整数，才能做加法遍历（`binary.BigEndian.Uint32`）
- 用 `map[string]struct{}` 存储已占用 IP，实现 O(1) 查重而不是 O(n) 线性扫描
